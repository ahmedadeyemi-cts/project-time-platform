# 020B Customer Directory Workflow

## Scope

020B strengthens the Customer Directory as a workflow-ready module that supports project intake, resource assignment, cost review, approval/export preparation, and audit visibility.

## Build Additions

- Customer summary metrics now distinguish active and inactive customers.
- Directory summary now identifies customer records that still need contact coverage.
- Directory summary now identifies customers with project or intake cost context.
- Search now includes customer name, customer code, contact name, contact email, and contact relationship.
- Selected customer detail now shows active/inactive state.
- Selected customer detail now includes workflow readiness checks for customer status, contact coverage, primary contact, cost context, and over-plan risk.
- Empty search results now show a clear user-facing message.

## Deferred Validation

Full browser validation is intentionally deferred until 020J. For 020B, the expected check is a frontend build only.
