# Module 001 mobile-mode design

Mobile mode is a presentation preference, not an authorization decision.

- Provide a visible checkbox or toggle labeled Mobile mode.
- Persist the preference locally and also respond automatically to narrow viewports.
- Use a single-column layout and touch targets of at least 44 pixels.
- Keep active timer, week context, totals, Save Draft, and Submit Week accessible.
- Collapse the activity sidebar into an expandable section or drawer.
- Replace wide view tiles with a compact selector or scrollable tab row.
- Keep all six Timesheet views available.
- Weekly Grid may use controlled horizontal scrolling or cards, but no data or action may disappear.
- Maintain keyboard navigation, visible labels, focus states, and screen-reader names.
- Do not create a separate mobile data model or application.
