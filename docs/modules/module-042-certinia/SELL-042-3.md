# Module 042.3 — SELL Commercial Read Model

Module 042.3 establishes the guarded commercial-source contract for ProjectPulse without enabling live SELL synchronization or changing invoice calculations prematurely.

## Current behavior

- Reads the existing project SELL quote number.
- Reads the active effective project/customer rate card and rate descriptions.
- Reads SELL connector readiness and last successful synchronization time.
- Classifies billing method as time and materials, fixed-fee milestone, hybrid, or non-billable.
- Surfaces milestone readiness without creating milestone records.
- Adds commercial-source status to the Invoice & Billing Center.
- Leaves invoice calculations on current stored rates by default.

## Guarded cutover

SELL becomes the invoice commercial source only when all conditions are true:

1. `PROJECTPULSE_SELL_COMMERCIAL_READ_MODEL_ACTIVE=true`.
2. The SELL connector is configured or connected.
3. Inbound synchronization is enabled.
4. A successful synchronization timestamp exists.
5. The project has a SELL quote number.
6. An active effective commercial rate card has billable rate lines.

The deployment sets both of these values to false:

- `PROJECTPULSE_SELL_LIVE_SYNC_ENABLED=false`
- `PROJECTPULSE_SELL_COMMERCIAL_READ_MODEL_ACTIVE=false`

## Persistence

This module uses existing canonical objects:

- `projects.sell_quote_number`
- `project_billing_profiles.default_rate_card_id`
- `work_rate_cards`
- `work_rate_card_lines`
- `external_integration_connections`

No database migration or schema modification is included.
