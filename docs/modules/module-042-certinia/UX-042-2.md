# Module 042.2 — Invoice Output and UX Correction

This forward change refines the deployed Module 042 Certinia delivery foundation without changing database schema, invoice numbers, source time, or existing immutable invoices.

## Delivered corrections

- Automatically loads the latest immutable invoice for a selected project.
- Enables top-level PDF and Excel downloads when an immutable invoice is selected.
- Adds independent customer-output controls for engineer, Project Manager, and Project Coordinator names.
- Hides all personal names by default across screen preview, PDF, Excel, and Certinia payloads.
- Displays currency-first rates such as `$210.00/hr`.
- Keeps Hours, Rate, Amount, and Date columns from wrapping.
- Allows tasks and work-performed descriptions to wrap naturally.
- Distinguishes fully invoiced projects from projects with no approved uninvoiced time.

## Commercial source direction

The current effective-rate foundation remains unchanged in this module. A later forward module will synchronize customer-facing project name, rate code, rate description, unit rate, currency, effective dates, contracted amount, and commercial revision from SELL into a ProjectPulse commercial read model. Immutable invoices will retain the synchronized SELL identifiers and values used at invoice creation time.

## Milestone direction

Milestone and hybrid billing are intentionally not added in this UX-only module. The future Work Register billing method should support:

- Time and Materials
- Fixed Fee — Milestone
- Hybrid — Time and Milestone
- Non-billable

Milestone items must include commercial amount or percentage, trigger/acceptance criteria, target date, approval, customer evidence, SELL quote line, GSD reference, previously invoiced amount, and remaining amount. Milestone invoice amounts must come from the approved commercial milestone rather than engineering hours.

## Safety

- No migration.
- No invoice created during deployment.
- No invoice number consumed.
- No Certinia transmission performed.
- `PROJECTPULSE_CERTINIA_ENABLED=false` remains enforced.
- Modules 057, 058, and 042.1 remain in history and are not replaced.
