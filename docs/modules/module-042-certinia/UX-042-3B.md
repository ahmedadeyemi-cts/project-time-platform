# Module 042.3B — Invoice Authorization and Native Excel

## Purpose

Correct inconsistent invoice-output authorization and replace CSV/HTML spreadsheet output with a native Excel Open XML workbook.

## Authorization

PDF, Excel, and Certinia invoice-status endpoints now use the same project-scope model as the Invoice & Billing Center:

- active Project Manager assignment;
- active Project Coordinator assignment;
- project assignment;
- canonical broad billing and administrative roles.

The canonical broad-role list includes Super Administrator, Administrator, Project Team Coordinator, Accounting, Accounting Billing, Billing, Finance, and Executive role codes, while preserving existing aliases.

## Native Excel output

The server produces a genuine `.xlsx` ZIP package with SpreadsheetML parts rather than HTML renamed as `.xls` or browser-generated CSV.

The workbook contains:

- an Invoice Summary worksheet;
- an Invoice Detail worksheet;
- native date, decimal, and currency cells;
- frozen headers;
- a formatted Excel table with filters and banded rows;
- wrapped task and work descriptions;
- customer-output privacy choices;
- invoice totals and print configuration.

## Safety

- No database schema change.
- No invoice is created.
- No invoice number is consumed.
- SELL synchronization and commercial cutover remain disabled.
- Certinia transmission remains disabled and is not performed.
