# Module 042.4 — SELL Inbound Synchronization and Commercial Snapshots

Module 042.4 adds controlled SELL commercial snapshot ingestion and project-level comparison without activating SELL as the invoice source.

## Persistence

Snapshots are stored immutably in the existing `external_integration_sync_runs.run_metadata_json` structure with `system_code=SELL` and `sync_mode=commercial_snapshot_comparison`. No schema migration is required. Duplicate payloads reuse a SHA-256 correlation key.

## Capabilities

- Quote number, version, revision, customer-facing project name, billing method, currency, contracted amount, and effective dates.
- Rate-line descriptions, categories, unit rates, and effective dates.
- Milestone definitions and contracted-total comparison.
- Source-payload SHA-256, import actor, timestamp, history, and idempotency.
- Comparison against current stored project and rate-card data.

## Guardrails

- `PROJECTPULSE_SELL_SNAPSHOT_IMPORT_ENABLED=true`
- `PROJECTPULSE_SELL_LIVE_SYNC_ENABLED=false`
- `PROJECTPULSE_SELL_COMMERCIAL_READ_MODEL_ACTIVE=false`
- `PROJECTPULSE_CERTINIA_ENABLED=false`
- Invoice calculations remain on current stored rates.
- No invoice creation, invoice-number consumption, or Certinia transmission.
