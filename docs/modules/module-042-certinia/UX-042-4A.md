# Module 042.4A - Branded Invoice Output Parity

Module 042.4A replaces the plain-text customer PDF and audit-oriented summary workbook with branded US Signal professional-services invoice artifacts.

## Customer-facing title

- Preview/source label: `US Signal SP Invoice Preview`
- Immutable PDF title: `US Signal SP Invoice`
- Printable Excel worksheet: `US Signal SP Invoice`
- Download names: `US-Signal-SP-Invoice-<invoice-number>.pdf` and `.xlsx`

## PDF design

- US Signal branded header and invoice number
- Customer, project, billing period, ownership, PO, Certinia, SELL, and Salesforce references
- Six-column billed-work table with wrapped descriptions and repeated headers
- Alternating row backgrounds, page numbering, invoice notes, privacy statement, and totals panel
- Standard PDF fonts only; no runtime browser or pop-up dependency

## Excel design

- `US Signal SP Invoice`: printable customer-facing invoice matching the PDF structure
- `Invoice Detail`: filterable operational detail with native dates, hours, rates, and amounts
- `Audit Metadata`: immutable snapshot hash and internal generation metadata kept away from the customer sheet

## Guardrails

- API-only source and deployment
- No web image build or deployment
- Module 001 worktree and branch are not changed
- Module 059 backend remains present
- Invoice source data and immutable ledger are unchanged
- No database schema change
- No invoice creation or invoice-number consumption
- SELL commercial cutover remains disabled
- Certinia transmission remains disabled
